# Do Eventer

イベントごとに必要な連絡・予定・参加者・料金をひとつにまとめる、iOS／Android共通のイベントコミュニケーションアプリです。LINEなどで友だち登録やグループ作成をせず、招待コードだけでイベント単位につながる体験を目指しています。

## 現在できること

- イベント一覧とカレンダー表示
- 単日・複数日に対応したイベント作成
- iOS／Android標準の日付・時刻選択
- イベント作成時の場所検索・現在地・地図タップによる位置指定
- 招待コードによる参加（試作用コード：`HAKONE26`）
- 日時、集合場所、説明、参加者の確認
- 日をまたぐタイムフローの表示
- 参加費、食事代、宿泊費、立替えなど複数の集金登録
- 集金ごとの立替者、対象者、均等割り／個別金額、支払い状況の管理
- OS標準の共有画面を使った招待
- QRコードによるイベント参加
- 端末カレンダーへのイベント追加
- イベント専用グループチャット
- 予定追加とプロフィール編集
- メール登録、ログイン、メール確認、パスワード再設定
- 参加承認、主催者／共同主催者／参加者の権限管理
- 参加・未定・不参加の回答と、主催者による参加申請の承認・拒否
- イベント開始と未払い集金期限の端末内リマインダー
- 通知タップから対象イベント・集金詳細への移動
- 主催者による候補日の追加と、参加者の ○・△・× 投票
- 主催者による共同主催者の任命・解除
- 投票した候補日を正式なイベント日程へ確定
- 入力画面のキーボード追従と、自己通報・自己ブロックの防止
- 通報、ブロック、アカウント削除、データ書き出しの基盤
- Supabase接続時のクラウド同期と、未接続時の端末内保存

現在は公開前のMVPです。Supabaseを設定しなくてもローカルモードで一通り試せ、設定した場合はアカウントごとのクラウドデータを利用できます。現金やPayPayなど支払方法を限定しないため、実際の送金・決済は行わず、支払状態だけを記録します。端末内リマインダーはExpo Goでも確認できます。予定変更やチャットを別端末へ届けるリモートプッシュ通知と、本番環境での権限テストは今後の開発項目です。

Expo Goの準備、日常のプログラミング、SDK更新、Development Build、ストア公開については[日本語開発ガイド](./DEVELOPMENT_GUIDE_JA.md)を参照してください。

## 採用技術

- Expo SDK 54
- React Native 0.81
- TypeScript
- Expo Router
- Supabase（Auth / PostgreSQL / Realtime / Edge Functions）
- Expo Notifications（端末内リマインダー）
- pnpm 11

SDK 54はReact Native 0.81、React 19.1、Node.js 20.19以上の組み合わせです。詳細は[Expo SDK 54公式リファレンス](https://docs.expo.dev/versions/v54.0.0/)を参照してください。

## 最初の起動

事前にNode.js 20.19以上と、スマートフォンへExpo Goをインストールしてください。

```sh
npx.cmd pnpm@11.9.0 install
npm.cmd run start
```

表示されたQRコードを読み取ると実機で確認できます。

- Android：Expo Goの「Scan QR code」から読み取る
- iPhone：標準カメラでQRコードを読み取る
- Webブラウザ：起動中の画面で `w` を押す
- Androidエミュレーター：`a` を押す（Android Studioが必要）
- iOSシミュレーター：`i` を押す（macOSとXcodeが必要）

現在のWindows環境では、`START_DO_EVENTER.cmd` をダブルクリックしても起動できます。`pnpm` が認識されない場合も、起動は `npm.cmd run start` を使用できます。

## 主な構成

```text
app/                 画面と画面遷移
  (tabs)/            ホーム、予定、マイページ
  event/[id].tsx     イベント詳細
  event/[id]/chat.tsx イベントチャット
  create.tsx         イベント作成
  join.tsx           招待コード参加
components/          共通表示部品
context/             試作版のイベント状態管理
data/                サンプルイベント
lib/                 Supabase接続とクラウド同期
supabase/            SQLマイグレーションとEdge Functions
types/               イベントのデータ型
constants/           色・文字・影などのデザイン設定
```

## セキュリティ設計

- すべてのユーザーデータテーブルでRow Level Securityを使用
- イベント参加状態と役割に応じた読み書き制御
- 招待コードはハッシュ化し、有効期限・利用回数・失効に対応
- Secret keyと`service_role`はクライアントへ含めない
- `.env`、証明書、署名鍵はGit管理対象外
- 通報、ブロック、アカウント削除、データ書き出しの導線を用意

設計と実装は公開前レビュー中です。本番公開前に、複数アカウントでのRLS自動テストと第三者によるセキュリティ確認を行います。
